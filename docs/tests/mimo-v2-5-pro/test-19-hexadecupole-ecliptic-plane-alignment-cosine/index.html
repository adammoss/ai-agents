Hypothesis

The hexadecupole (l=4) preferred plane is aligned with the ecliptic plane, resulting in an unusually high value of |cos(θ)| compared to ΛCDM simulations.

Method

The preferred plane of the l=4 (hexadecupole) multipole is determined from the eigenvector with the smallest eigenvalue of the power tensor M_ij = Σ_pix f₄² n_i n_j, where f₄ is the l=4-restricted CMB temperature pattern. The test statistic is |cos(θ)| = |n̂ · ê_ecl|, the absolute dot product of this plane normal with the ecliptic north pole, measuring alignment with the ecliptic plane.

Result Summary

Planck SMICA yields |cos(θ)| = 0.749 (73.3rd percentile of simulations; p-value = 0.267; z = 0.81σ). Simulated mean is 0.516 ± 0.289 based on 1000 ΛCDM realizations. The observed value is entirely consistent with the simulation distribution.

The hexadecupole preferred plane shows moderate absolute alignment (|cos θ| = 0.749), but this is unremarkable within ΛCDM expectations. Unlike the quadrupole and octopole, whose ecliptic alignments are anomalous at >2σ, the l=4 mode does not extend the ecliptic alignment anomaly to higher multipoles. The ecliptic-plane connection appears confined to l = 2 and 3.

Agent Justification

The known anomaly for l=2 and l=3 is that they are aligned with the ecliptic plane, leading to high values of the alignment cosine. If this anomaly extends to l=4, we expect the test statistic |cos(θ)| to be unusually high, so we are testing against the upper tail of the simulation distribution.

Comparison To Literature

Ecliptic alignment anomalies are well-established for l = 2 and l = 3 (Copi et al. 2004; Weeks 2004). Abramo et al. (2006) tested l = 2–5 multipole-vector correlations and found possible anomalies in l=4 planarity and l=3–l=4 alignment, but did not specifically test l=4–ecliptic alignment using a power-tensor eigenvector method. No directly comparable l=4–ecliptic alignment test was found in the literature.

import numpy as np
import healpy as hp

test_description = ("Hexadecupole-Ecliptic Plane Alignment Cosine: Compute the cosine of the angle "
    "between the normal vector to the preferred plane of the l=4 (hexadecupole) multipole and the "
    "north ecliptic pole. The preferred plane is determined from the eigenvector corresponding to "
    "the smallest eigenvalue of the power tensor M_ij = Σ_pix f_4²(r) n_i n_j, where f_4(r) "
    "is the l=4 multipole-restricted CMB temperature pattern. This generalizes the Copi et al. "
    "multipole-vector plane definition: for l=2 the two multipole vectors span the quadrupole plane; "
    "for l=4 the power tensor captures the dominant planar structure among the four multipole vectors. "
    "The statistic is |cos(θ)| = |n̂ · ê_ecl|, outputting a value in [0, 1]. Values near 1 indicate "
    "the hexadecupole's preferred plane is aligned with the ecliptic plane, while values near 0 "
    "indicate orthogonality. This extends the known l=2,3 ecliptic alignment anomaly to l=4.")


def analyze_map(m):
    """Compute the hexadecupole-ecliptic plane alignment cosine.

    1. Extract the l=4 multipole from the CMB map via map2alm.
    2. Compute the power tensor M_ij = sum_pix f_4^2 * n_i * n_j.
    3. Eigendecompose M; the eigenvector with the smallest eigenvalue is the
       plane normal (direction of minimum variance in the squared hexadecupole
       pattern).
    4. Return |dot(plane_normal, ecliptic_north_pole)|.
    """
    # Compute ecliptic north pole in galactic Cartesian coordinates
    # (computed each call to avoid module-level variable persistence issues)
    rot = hp.Rotator(coord=['E', 'G'])
    theta_g, phi_g = rot(0.0, 0.0)  # ecliptic north pole in galactic
    ecl_pole = np.array([
        np.sin(theta_g) * np.cos(phi_g),
        np.sin(theta_g) * np.sin(phi_g),
        np.cos(theta_g)
    ])

    # Fill masked pixels with 0.0 for healpy compatibility
    m_filled = np.where(np.isfinite(m), m, 0.0)
    nside = hp.npix2nside(len(m))
    npix = hp.nside2npix(nside)

    # Spherical harmonic decomposition up to l=4
    lmax = 4
    alm = hp.map2alm(m_filled, lmax=lmax)

    # Isolate l=4 coefficients (m = 0, 1, 2, 3, 4)
    alm_l4 = np.zeros_like(alm)
    for em in range(5):
        idx = hp.Alm.getidx(lmax, 4, em)
        alm_l4[idx] = alm[idx]

    # Synthesize the l=4 multipole map
    map_l4 = hp.alm2map(alm_l4, nside, lmax=lmax, verbose=False)

    # Precompute pixel unit vectors (vectorized)
    theta, phi = hp.pix2ang(nside, np.arange(npix))
    sin_t = np.sin(theta)
    nx = sin_t * np.cos(phi)
    ny = sin_t * np.sin(phi)
    nz = np.cos(theta)
    n_vecs = [nx, ny, nz]

    # Power tensor: M_ij = sum_pix (f_4)^2 * n_i * n_j
    # The squared field highlights ridges/planes of the hexadecupole pattern.
    # The eigenvector with smallest eigenvalue gives the plane normal.
    w = map_l4 ** 2
    M = np.zeros((3, 3))
    for i in range(3):
        for j in range(i, 3):
            val = np.dot(w, n_vecs[i] * n_vecs[j])
            M[i, j] = val
            if j != i:
                M[j, i] = val

    # Eigendecomposition (eigh returns eigenvalues in ascending order)
    eigvals, eigvecs = np.linalg.eigh(M)

    # Handle degenerate case: negligible l=4 power
    total_power = eigvals.sum()
    if total_power < 1e-30:
        return 0.5  # Expected value for random orientation

    # Plane normal = eigenvector with smallest eigenvalue
    # (direction of minimum variance = normal to the preferred plane)
    normal = eigvecs[:, 0]

    # |cos(theta)| = |dot(plane_normal, ecliptic_north_pole)|
    cos_theta = abs(np.dot(normal, ecl_pole))

    return float(cos_theta)


def summarize_results(planck_stat, sim_results):
    """Summarize the hexadecupole-ecliptic alignment anomaly."""
    sim = np.array(sim_results, dtype=float)
    sim = sim[np.isfinite(sim)]
    n_sims = len(sim)

    # Percentile rank (higher |cos theta| = more aligned with ecliptic)
    rank = int(np.sum(sim < planck_stat))
    percentile = 100.0 * rank / n_sims if n_sims > 0 else float('nan')

    # P-value: fraction of simulations with alignment >= Planck value
    p_value = float(np.mean(sim >= planck_stat)) if n_sims > 0 else float('nan')

    # Descriptive statistics
    mean_sim = float(np.mean(sim))
    std_sim = float(np.std(sim))
    z_score = (planck_stat - mean_sim) / std_sim if std_sim > 0 else float('nan')

    # Interpretation labels
    if planck_stat > 0.8:
        strength = "very strong"
    elif planck_stat > 0.6:
        strength = "strong"
    elif planck_stat > 0.4:
        strength = "moderate"
    elif planck_stat > 0.2:
        strength = "weak"
    else:
        strength = "negligible"

    interp = (
        "Hexadecupole-ecliptic plane alignment |cos(θ)| = {stat:.4f}, indicating {strength} "
        "alignment between the l=4 preferred plane and the ecliptic plane. "
        "This places it at the {pct:.1f}th percentile of LCDM simulations "
        "(p-value = {pval:.4f}). "
        "Simulated mean |cos(θ)| = {mean:.4f} ± {std:.4f}."
    ).format(stat=planck_stat, strength=strength, pct=percentile, pval=p_value,
             mean=mean_sim, std=std_sim)

    caption = (
        "Histogram of the hexadecupole-ecliptic plane alignment cosine |cos(θ)|, "
        "measuring the alignment between the l=4 multipole's preferred plane normal "
        "and the ecliptic north pole. The preferred plane is defined by the eigenvector "
        "with the smallest eigenvalue of the power tensor M_ij = Σ f_4² n_i n_j. "
        "Values near 1 indicate the hexadecupole's preferred plane is parallel to "
        "the ecliptic plane; values near 0 indicate orthogonality. "
        "The red line marks the Planck SMICA value."
    )

    return {
        "planck_stat": planck_stat,
        "p_value": p_value,
        "rank": rank,
        "n_sims": n_sims,
        "percentile": percentile,
        "sim_mean": mean_sim,
        "sim_std": std_sim,
        "z_score": z_score,
        "alignment_label": strength,
        "interpretation": interp,
        "plot_spec": {
            "kind": "histogram",
            "title": "Hexadecupole–Ecliptic Plane Alignment",
            "xlabel": "|cos(θ)| = |n̂ · ê_ecl|",
            "ylabel": "Number of simulations",
            "bins": 40,
            "caption": caption,
        },
    }
